# VASP输出格式问题——浮点数报错

outcar文件原子核静电势部分，由于列宽压缩导致大体系时字段会粘连，数据处理时`split()` 无法正确拆分，脚本读取 float 时就会报错。

```python
 average (electrostatic) potential at core
  the test charge radii are     1.0894  0.6991  0.7089  0.5201  1.0894  1.1875
  (the norm of the test charge is              1.0000)
       1 -104.2504       2 -54.8687       3 -54.8687       4 -54.9332       5 -54.9332
...
       131-104.3951     132-104.3343     133-104.4251     134-104.3429     135-104.3953
       136-104.3344     137-104.4254     138-104.3406     139-104.4365     140-104.3431
       141-104.3909     142-104.3411     143-104.4364     144-104.3432     145-104.3906
...
```

数据处理时导入了`extract_potential_at_core_vasp`函数提取电势值。

```python
from spinney.io.vasp import extract_potential_at_core_vasp
```

删除导入，采用正则法重新定义`extract_potential_at_core_vasp`函数。

```python
        def extract_potential_at_core_vasp(outcar_path):
            potentials = []
```
